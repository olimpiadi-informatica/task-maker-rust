1//! The `italian_yaml` format is defined by [`cms`](https://cms.readthedocs.io/en/v1.4/External%20contest%20formats.html#italian-import-format)
2//! and it's the most used format in the Italian Olympiads.
3//!
4//! # `gen/GEN` format
5//! Here it's provided the definition of the format of the `gen/GEN` file, as interpreted by
6//! task-maker. The aim is to be as compatible as possible to the format accepted by `cmsMake` and
7//! `cmsImportTask`.
8//!
9//! The `gen/GEN` file describes how the input files of the testcases should be generated and how
10//! the subtasks are composed. The formal definition of the format can be found looking at the
11//! [parsing grammar](https://github.com/olimpiadi-informatica/task-maker-rust/blob/master/task-maker-format/src/ioi/format/italian_yaml/GEN.pest).
12//! The format is described here informally.
13//!
14//! Each line of that file can be of one of the following types:
15//!
16//! - if the line starts with `# ` (number sign followed by a space) it is a comment and will be
17//!   ignored. Example: `# this is a comment`
18//! - if the line starts with `#ST:` it is the marker of the start of a new subtask. Following the
19//!   column there is the score of the subtask, an integer number. Example: `#ST: 20`, meaning that
20//!   all the following testcases, before the next `#ST:` are grouped in a single subtask worth 20
21//!   points.
22//! - if the line starts with `#COPY:` it is a testcase with a static input file, meaning that the
23//!   input file will be simply copied from the path specified after the column. Example:
24//!   `#COPY: gen/hardcoded.in`. The path is relative to the task root directory and the file must
25//!   exists and be readable from task-maker. The path should not contain spaces.
26//! - non-empty lines not starting with `#` defines a new testcases each. Each line contains
27//!   command line arguments to pass to the generator executable. The generator should be named
28//!   `gen/generator.*` or `gen/generatore.*`. Example: `1 2 3`, the generator will be invoked
29//!   passing the three arguments.
30//!
31//! If a line contains a `#`, all the characters following it (`#` included) will be ignored as they
32//! are considered comments. Example: `1 2 3 # inline comment`.
33//!
34//! If no `#ST` lines are present, a single subtask worth 100 points is automatically added.
35//!
36//! ## Full example of `gen/GEN`
37//!
38//! ```text
39//! # N     M       seed
40//! #ST: 0
41//! #COPY: gen/example1.txt
42//! #COPY: gen/example2.txt
43//!
44//! #ST: 30
45//! 10      1000    101
46//! 20      1000    102
47//! 1       1       103 # corner case!
48//!
49//! #ST: 70
50//! 1000    10000   201
51//! 2000    10000   202
52//! ```
53//!
54//! In this example the first line is ignored since it's a comment, then the definition of a subtask
55//! of zero points starts. The content of that subtask are 2 hardcoded input files, for example the
56//! sample cases in the text statement.
57//!
58//! Then a new subtask of 30 points starts, it's composed of 3 testcases. Note that the comment on
59//! the third will be ignored and just the 3 numbers will be passed to the generator.
60//!
61//! Then there is a third subtask with 2 testcases.
62//!
63//! # `gen/cases.gen` format
64//!
65//! The `gen/GEN` format is pretty limited regarding some important aspects of task preparation. For
66//! example it allows you to use just a single generator and a single validator. A new format, not
67//! yet officially supported by `cms` (but workaround exists!), is here described.
68//!
69//! The formal definition of the format can be found in the
70//! [parsing grammar](https://github.com/olimpiadi-informatica/task-maker-rust/blob/master/task-maker-format/src/ioi/format/italian_yaml/cases.gen.pest).
71//! An informal explanation is here provided.
72//!
73//! Similarly to `gen/GEN`, each line is independent and can be one of the following:
74//! - if it starts with `#` it is a comment and will be ignored.
75//! - if it starts with `:` it is a command and what follows is described below.
76//! - otherwise if it does not start with `#` nor `:`, it's a simple testcase definition. The
77//!   testcases will be generated using the _current generator_, which is either the default one or
78//!   the overridden one in the current subtask.
79//!
80//! If a line contains a `#`, all the characters following it (`#` included) will be ignored as they
81//! are considered comments. Example: `1 2 3 # inline comment`.
82//!
83//! ## Backwards compatibility with cms
84//! Since the importers of cms do not yet support the `cases.gen` format, there is a workaround that
85//! works as follows: when the task is built task-maker will create a _fake_ `gen/GEN` with just the
86//! metadata needed by `cmsImportTask` (like the number of testcases and subtasks).
87//!
88//! This file also contains some comments that may be found useful for debugging the generations.
89//!
90//! That file will be ignore by task-maker if the `cases.gen` file is present and will be removed
91//! with `--clean`. Note that you should not edit that file because it will be overwritten the next
92//! time task-maker will be launched. To keep that file you should remove the comment containing
93//! `tm-allow-delete`.
94//!
95//! ## `cases.gen` commands
96//! The lines starting with a column `:` are commands. What follows the column is the actual command
97//! which can be of many types.
98//!
99//! ### `: GEN name path [args...]`
100//! This command registers a new generator for the task. The generator is referenced in the file
101//! with the specified `name`, a unique string without spaces. That generator's source file will be
102//! found at the specified `path`. The optional following `args...` define the command line
103//! arguments of this generator, they will be used for validating the constraints (see below).
104//!
105//! If the name is `default` the generator will be considered the default one and will be used when
106//! no specific generator is selected.
107//!
108//! Example: `: GEN default gen/generator.py N M seed` defines the _default_ generator for the task,
109//! whose source file is at `gen/generator.py` (relative to the task's root directory) and will
110//! accept 3 arguments named `N`, `M` and `seed`. When this generator will be used the variables
111//! `$N`, `$M` and `$seed` will be set and available for the constraint evaluation.
112//!
113//! Example: `: GEN line gen/line.py` defines a new generator named `line` whose parameters are not
114//! known and won't be validated.
115//!
116//! ### `: VAL name path [args...]`
117//! This command registers a new validator for the task. The semantics of the command are the same
118//! of `: GEN name path [args...]`, including the behaviour of the `default` name.
119//!
120//! If no arguments are specified for the validator the default behaviour is to pass the variables
121//! `$INPUT` and `$ST_NUM` (similar to `gen/GEN`, but the subtask is 0-based).
122//!
123//! Note that, differently than `: GEN` the arguments of the validator do not define new variables,
124//! instead defines which parameters task-maker will pass to the validator. Because of that the
125//! variables should be prefixed with `$` (the variables are used, not declared).
126//!
127//! Note that the `name` must be unique among the validators, but it can be the same of the one of
128//! a generator.
129//!
130//! Example: `: VAL default gen/validator.py` defines the _default_ validator for the task.
131//!
132//! Example: `: VAL line gen/val_line.py $INPUT $ST_NUM` defines the `line` validator which takes 2
133//! arguments, the same as the default behaviour, but using the variables to specify them.
134//!
135//! ### `: GEN name`
136//! This command overrides the default generator for the current subtask, meaning that all the
137//! testcases in the current subtask, following this command, will use the generator named `name` by
138//! default. A generator named `name` must have been previously defined.
139//!
140//! Example: `: GEN line` sets the current generator to `line`.
141//!
142//! ### `: VAL name`
143//! This command overrides the default validator for the current subtask, meaning that all the
144//! testcases in the current subtask, following this command, will use the validator named `name`.
145//! A validator named `name` must have been previously defined.
146//!
147//! Example: `: VAL line` sets the current validator to `line`.
148//!
149//! ### `: CONSTRAINT operand (operator operand)+`
150//! This command adds a constraint that validates the parameters of the testcases. The arguments of
151//! `: CONSTRAINT` form an expression that is an inequality (with equalities allowed) between
152//! constants and variables. When a testcase is defined using a generator with the arguments known,
153//! all those variables become defined and will be checked with all the constraints.
154//!
155//! The operators available are: < <= > >= =, but note that the inequalities must have the same
156//! direction (cannot mix < and >).
157//!
158//! Constraints defined before the first subtask will be used for all the subtasks. Constraints
159//! defined inside a subtask will be used only for that subtask.
160//!
161//! Example: `:CONSTRAINT 0 <= $N < $M <= 1000000` will check that the variables `$N` and `$M` are
162//! between 0 and 1000000 and `$N` is smaller than `$M`.
163//!
164//! ### `: SUBTASK score [name]`
165//! This command marks the start of a new subtask, just like how `#ST` in `gen/GEN` did. The score
166//! can be a simple floating point number (either an integer or an integer.integer). The name
167//! that follows is optional and will be included in the subtask metadata.
168//!
169//! When a new subtask is started the generator and validator will be reset to the default ones.
170//!
171//! Example: `: SUBTASK 40 all-the-nodes-are-in-a-line` defines a new subtask worth 40 points, with
172//! the provided name.
173//!
174//! ### `: COPY path`
175//! This command creates a new testcase coping the input file from the specified path, relative to
176//! the task root directory. The file will be validated using the current validator of the subtask.
177//!
178//! Example: `: COPY gen/hardcoded.in`
179//!
180//! ### `: RUN name args...`
181//! This command creates a new testcase using the generator named `name`, passing to it the
182//! following arguments. The generator must have been previously defined.
183//!
184//! If the generator has the definition of its parameters, they will be used for assigning the
185//! variables for checking the constraints. All the constraints must be satisfied for each testcase.
186//!
187//! The arguments provided are parsed with a shell lexer, meaning that `"` and `'` have a semantic
188//! value (the same as a shell). Unlike `gen/GEN` you can pass arguments with a space in it using
189//! the quotes.
190//!
191//! Example: `: RUN line 1 2 3` will run the `line` generator passing the three integers as
192//! arguments.
193//!
194//! ## Testcase definition
195//! Similarly to `gen/GEN` lines that are not commands nor comments are simple testcase definition.
196//! Their semantics is the same of `: RUN default args...`.
197//!
198//! ## Automatic variables
199//! In the constraint evaluation and in the validator argument specification all the variables
200//! obtained from the parsing of the generator's arguments will be available. Also some automatic
201//! variables will be available:
202//! - `$ST_NUM`: the 0-based index of the subtask
203//! - `$ST_NAME`: the name of the subtask
204//! - `$INPUT` _(only for validators)_: the name of the file to validate
205//!
206//! ## Full example of `cases.gen`
207//! ```text
208//! : GEN default gen/generator.py N M seed
209//! : GEN line gen/line.py N seed
210//! : GEN hard gen/hard.py
211//!
212//! : VAL default gen/validator.py
213//! : VAL line gen/val_line.py $INPUT $ST_NUM # same as default
214//!
215//! : CONSTRAINT 1 <= $N <= 1000
216//! : CONSTRAINT 1 <= $M <= 1000000
217//!
218//! : SUBTASK 0 examples
219//! : COPY gen/example1.in
220//! : COPY gen/example2.in
221//!
222//! : SUBTASK 30 nodes-are-in-a-line
223//! : GEN line
224//! : VAL line
225//! : CONSTRAINT $N <= 500
226//! 500   101
227//! 500   102
228//!
229//! : SUBTASK 70
230//! 1000   1000      201
231//! 1000   1000000   202
232//! : RUN hard 1000 1000 95% 12.3 203
233//! ```
234//!
235//! In this example 3 generators and 2 validators are defined, named `default`, `line`, `hard` and
236//! `default`, `line` respectively. The `default` and `line` generators have their parameters
237//! specified, they will be used for the constraints check; the `hard` generator arguments won't be
238//! validated.
239//!
240//! Note that the second validator has the arguments specified, and they are the same as the default
241//! ones. Also note that the inline comment will be ignored.
242//!
243//! This file defines 3 subtasks, worth 0, 30 and 70 points each.
244//!
245//! The first subtask contains 2 testcases whose file will be simply copied from the specified
246//! paths.
247//!
248//! The second subtask has 2 testcases each generated with the `line` generator and validated with
249//! the `line` validator. Note that there is an additional constraint for the subtask, it will be
250//! checked only in this subtask.
251//!
252//! The third subtask will use the `default` generator and validator, except for the last testcase
253//! which will use the `hard` one. Note that since the `hard` generator does not have the argument
254//! specification, its parameters won't be checked. Also note that the constraint `$N <= 500` won't
255//! be checked because it was scoped only to the second subtask.
256//! The subtask also does not have a name, the default one (`subtask2`) will be used.

321fn cms_serialize_score_type<S>(

324) -> Result<S::Ok, S::Error>

326    S: Serializer,

328    match val {

335/// Deserialized data from the task.yaml of a IOI format task.
336#[derive(Debug, Serialize, Deserialize)]
337struct TaskYAML {
338    /// The name of the task (the short one).
339    #[serde(alias = "nome_breve")]
340    pub name: String,
341    /// The title of the task (the long one).
342    #[serde(alias = "nome")]
343    pub title: String,
344    /// The score type to use for this task.
345    #[serde(serialize_with = "cms_serialize_score_type")]
346    pub score_type: Option<TestcaseScoreAggregator>,
347    /// The parameters of the score type.
348    #[serde(skip_serializing_if = "Option::is_none")]
349    pub score_type_parameters: Option<Vec<(f64, String)>>,
350    /// The number of decimal digits when displaying the scores.
351    #[serde(default)]
352    pub score_precision: usize,
353    /// The primary language used when importing the task.
354    pub primary_language: Option<String>,
355
356    /// The time limit for the execution of the solutions, if not set it's unlimited.
357    #[serde(alias = "timeout")]
358    pub time_limit: Option<f64>,
359    /// The memory limit in MiB of the execution of the solution, if not set it's unlimited.
360    #[serde(alias = "memlimit")]
361    pub memory_limit: Option<u64>,
362
363    /// Whether this is an output only task. Defaults to false.
364    #[serde(default)]
365    pub output_only: bool,
366    /// The input file for the solutions, usually 'input.txt' or '' (stdin). Defaults to `''`.
367    #[serde(default = "default_infile")]
368    pub infile: String,
369    /// The output file for the solutions, usually 'output.txt' or '' (stdout). Defaults to `''`.
370    #[serde(default = "default_outfile")]
371    pub outfile: String,
372
373    /// An integer that defines the difficulty of the task. Used only in booklet compilations.
374    pub difficulty: Option<u8>,
375    /// An integer that defines the level inside a _syllabus_ (for example for the Olympiads in
376    /// Teams). Used only in booklet compilations.
377    pub syllabuslevel: Option<u8>,
378
379    /// Number of solution processes to spawn in parallel in a communication task.
380    pub num_processes: Option<u8>,
381    /// The type of communication for the solution in a communication task.
382    ///
383    /// Can be either "std_io" for using stdin/stdout, or "fifo_io" for using pipes given in argv.
384    /// Defaults to "fifo_io".
385    #[serde(default = "UserIo::fifo_io")]
386    pub user_io: UserIo,
387
388    /// Compatibility with cms, unused.
389    pub score_mode: Option<String>,
390    /// Compatibility with cms, unused.
391    pub token_mode: Option<String>,
392    /// Compatibility with cms, unused.
393    pub public_testcases: Option<String>,
394    /// Compatibility with cms, unused.
395    pub feedback_level: Option<String>,
396}

481/// Given the path to the task directory, try to parse the task inside of it assuming the format is
482/// `italian_yaml`.
483///
484/// `italian_yaml` format is structured as follow:
485/// * `task.yaml.orig` or `task.yaml` - file with the task information
486/// * `gen/` - folder with the generator and validator
487///     * `generator.xxx` (also `generatore`)
488///     * `validator.xxx` (also `valida`)
489///     * `GEN` - subtask and testcase specifications
490/// * `sol/` - folder with solutions, graders and stubs
491///     * `solution.xxx` the official solution (also `soluzione`)
492///     * other solutions with different names
493/// * `check/` - folder with the checker (also `cor/`)
494///     * `checker.xxx` (also `correttore`)
495/// * `input/` - folder with the input files
496/// * `output/` - folder with the output files
497/// * `statement/` - folder with the statement (also `testo`)
498///
499/// A task must have a generator (and a GEN file) or the input files should be  put in `input/`.
500/// The official solution must be present or the output files should be put in `output/`.
501pub fn parse_task<P: AsRef<Path>>(
502    task_dir: P,
503    eval_config: &EvaluationConfig,
504) -> Result<IOITask, Error> {
505    let task_dir = task_dir.as_ref();
506
507    let task_yaml_path = task_dir.join("task.yaml");
508    let task_yaml_orig_path = task_dir.join("task.yaml.orig");
509    let task_yaml_overwrite: bool;
510    let mut yaml: TaskYAML;
511    if task_yaml_orig_path.exists() {
512        task_yaml_overwrite = true;
513        let file = File::open(&task_yaml_orig_path).with_context(|| {
514            format!(
515                "Cannot open task.yaml.orig from {}",
516                task_yaml_orig_path.display()
517            )
518        })?;
519        let yaml_orig: TaskYAMLOrig =
520            serde_yaml::from_reader(file).context("Failed to deserialize task.yaml.orig")?;
521        yaml = yaml_orig.into_task_yaml(task_dir);
522    } else if task_yaml_path.exists() {
523        task_yaml_overwrite = false;
524        let file = File::open(&task_yaml_path)
525            .with_context(|| format!("Cannot open task.yaml from {}", task_yaml_path.display()))?;
526        yaml = serde_yaml::from_reader(file).context("Failed to deserialize task.yaml")?;
527    } else {
528        bail!("No task.yaml found in {}", task_dir.display());
529    }
530    debug!("The yaml is {:#?}", yaml);
531
532    let map_file = |file: String| -> Option<PathBuf> {
533        match file.as_ref() {
534            "" => None,
535            _ => Some(file.into()),
536        }
537    };
538    let infile = map_file(yaml.infile.clone());
539    let outfile = map_file(yaml.outfile.clone());
540
541    let graders = list_files(task_dir, vec!["sol/grader.*", "sol/stub.*"]);
542    let grader_map = Arc::new(GraderMap::new(graders));
543    debug!("The graders are: {:#?}", grader_map);
544
545    let task_type = if let Some(comm) = parse_communication_task_data(task_dir, &yaml)? {
546        comm
547    } else {
548        parse_batch_task_data(task_dir, grader_map.clone())?
549    };
550
551    let gen_gen = task_dir.join("gen").join("GEN");
552    let cases_gen = task_dir.join("gen").join("cases.gen");
553    let output_generator: Box<dyn Fn(TestcaseId) -> OutputGenerator> =
554        if let TaskType::Batch(_) = &task_type {
555            Box::new(
556                detect_output_generator(task_dir.to_path_buf(), grader_map.clone())
557                    .context("Failed to detect output generator")?,
558            )
559        } else {
560            Box::new(|_| OutputGenerator::NotAvailable)
561        };
562
563    let inputs = if cases_gen.exists() {
564        debug!("Parsing testcases from gen/cases.gen");
565        let gen = cases_gen::CasesGen::new(&cases_gen, output_generator)?;
566        if !eval_config.dry_run {
567            gen.write_gen_gen().context("Failed to write gen/GEN")?;
568        }
569        gen.get_task_entries()
570    } else if gen_gen.exists() {
571        debug!("Parsing testcases from gen/GEN");
572        gen_gen::parse_gen_gen(
573            &gen_gen,
574            detect_validator(task_dir.into()).context("Failed to detect validator")?,
575            output_generator,
576        )?
577    } else {
578        debug!("Using testcases inside input/");
579        static_inputs::static_inputs(
580            task_dir,
581            detect_validator(task_dir.into()).context("Failed to detect validator")?,
582            output_generator,
583        )
584        .collect()
585    };
586
587    let mut subtasks = HashMap::new();
588    let mut testcases = HashMap::new();
589    let mut last_subtask: Option<SubtaskInfo> = None;
590    for input in inputs {
591        match input {
592            TaskInputEntry::Subtask(subtask) => {
593                if let Some(last_subtask) = last_subtask.take() {
594                    subtasks.insert(last_subtask.id, last_subtask);
595                }
596                last_subtask = Some(subtask);
597            }
598            TaskInputEntry::Testcase(testcase) => {
599                let st = last_subtask.as_mut().context("Testcase before Subtask")?;
600                st.testcases.push(testcase.id);
601                st.testcases_owned.push(testcase.id);
602                testcases.insert(testcase.id, testcase);
603            }
604        }
605    }
606    // insert the last subtask to the map
607    if let Some(subtask) = last_subtask.take() {
608        subtasks.insert(subtask.id, subtask);
609    }
610
611    loop {
612        let mut updated = false;
613        for st1_id in subtasks.keys().copied().collect_vec() {
614            let deps = subtasks.get(&st1_id).unwrap().dependencies.clone();
615            for st2_id in deps {
616                let ext = subtasks.get(&st2_id).unwrap().testcases.clone();
617                let into = subtasks.get_mut(&st1_id).unwrap();
618                let before = into.testcases.len();
619                into.testcases.extend(ext);
620                into.testcases.sort();
621                into.testcases.dedup();
622                let after = into.testcases.len();
623                if before != after {
624                    updated = true;
625                }
626            }
627        }
628        if !updated {
629            break;
630        }
631    }
632
633    let testcase_score_aggregator = yaml.score_type.unwrap_or(if subtasks.len() == 1 {
634        TestcaseScoreAggregator::Sum
635    } else {
636        TestcaseScoreAggregator::Min
637    });
638    yaml.score_type = Some(testcase_score_aggregator);
639
640    if task_yaml_overwrite {
641        if !task_yaml_path.exists() || is_tm_deletable(&task_yaml_path)? {
642            if !eval_config.dry_run {
643                yaml.score_type_parameters = Some(
644                    subtasks
645                        .iter()
646                        .sorted_by_key(|(id, _)| *id)
647                        .map(|(_, st)| {
648                            let testcases = st
649                                .testcases
650                                .iter()
651                                .map(|tc_num| format!("{tc_num:03}"))
652                                .join("|");
653                            (st.max_score, testcases)
654                        })
655                        .collect(),
656                );
657
658                let file = File::create(&task_yaml_path).with_context(|| {
659                    format!("Cannot open task.yaml from {}", task_yaml_path.display())
660                })?;
661                let mut buf_file = BufWriter::new(file);
662                writeln!(buf_file, "# Generated by task-maker. Do not edit!")?;
663                writeln!(buf_file, "# {}", TM_ALLOW_DELETE_COOKIE)?;
664                writeln!(buf_file, "# Removing or changing the line above will prevent task-maker from touching this file again.")?;
665                serde_yaml::to_writer(buf_file, &yaml).context("Failed to serialize task.yaml")?;
666            }
667        } else {
668            warn!(
669                "The task.yaml file does not contain {}. Won't overwrite",
670                TM_ALLOW_DELETE_COOKIE
671            );
672        }
673    } else if subtasks.values().any(|st| !st.dependencies.is_empty()) {
674        bail!("Use task.yaml.orig to use subtask dependencies");
675    }
676
677    let mut task = IOITask {
678        path: task_dir.into(),
679        task_type,
680        name: yaml.name,
681        title: yaml.title,
682        time_limit: yaml.time_limit,
683        memory_limit: yaml.memory_limit,
684        infile,
685        outfile,
686        testcase_score_aggregator,
687        score_precision: yaml.score_precision,
688        subtasks,
689        testcases,
690        grader_map,
691        booklets: Vec::new(),
692        difficulty: yaml.difficulty,
693        syllabus_level: yaml.syllabuslevel,
694        sanity_checks: Arc::new(get_sanity_checks(
695            &eval_config
696                .disabled_sanity_checks
697                .iter()
698                .map(String::as_str)
699                .collect::<Vec<_>>(),
700        )),
701        input_validator_generator: InputValidatorGenerator::new(
702            detect_validator(task_dir.to_path_buf()).context("Failed to detect validator")?,
703        ),
704    };
705    // split the creation of the task because make_booklets need an instance of Task
706    if !eval_config.no_statement {
707        task.booklets =
708            make_task_booklets(&task, eval_config).context("Failed to make booklets")?;
709    }
710    Ok(task)
711}

713/// Check if the file is deletable, i.e. it contains the TM_ALLOW_DELETE_COOKIE
714/// Assumes the file exists.
715pub(crate) fn is_tm_deletable(path: &Path) -> Result<bool, Error> {
716    let buffer = fs::read(path).with_context(|| format!("Cannot read file {}", path.display()))?;
717    let content = String::from_utf8_lossy(&buffer);
718    Ok(content.contains(TM_ALLOW_DELETE_COOKIE))
719}

721/// Search for a valid input validator inside the task directory. Will return a function that, given
722/// a subtask id, returns an `InputValidator` using that validator. If no validator is found,
723/// `InputValidator::AssumeValid` is used.
724fn detect_validator(
725    task_dir: PathBuf,
726) -> Result<impl Fn(Option<SubtaskId>) -> InputValidator, Error> {
727    let mut validators = find_source_file(
728        &task_dir,
729        vec![
730            "gen/validator.*",
731            "gen/valida.*",
732            "gen/validator",
733            "gen/valida",
734        ],
735        &task_dir,
736        "Input file validator at",
737        None,
738        WriteBinTo::path("bin/validator"),
739    );
740    if validators.len() > 1 {
741        let paths = validators.iter().map(|s| s.name()).collect::<Vec<_>>();
742        bail!("Multiple validators found: {:?}", paths);
743    }
744    let validator = validators.pop().map(Arc::new);
745    debug!("Detected input validator: {:?}", validator);
746    Ok(move |st: Option<SubtaskId>| -> InputValidator {
747        if let Some(validator) = validator.as_ref() {
748            InputValidator::Custom(
749                validator.clone(),
750                // for legacy support reasons the subtask is passed 1-based
751                vec![
752                    TM_VALIDATION_FILE_NAME.to_string(),
753                    st.map(|x| x + 1).unwrap_or(0).to_string(),
754                ],
755            )
756        } else {
757            InputValidator::AssumeValid
758        }
759    })
760}

762/// Search for a valid output generator (aka official solution) inside the task directory. Will
763/// return a function that, given a testcase id, returns an `OutputGenerator` using that generator.
764/// If no generator is found, `OutputGenerator::StaticFile` is used instead.
765fn detect_output_generator(
766    task_dir: PathBuf,
767    grader_map: Arc<GraderMap>,
768) -> Result<impl Fn(TestcaseId) -> OutputGenerator, Error> {
769    let mut official_solutions = find_source_file(
770        &task_dir,
771        vec![
772            "sol/solution.*",
773            "sol/soluzione.*",
774            "sol/solution",
775            "sol/soluzione",
776        ],
777        &task_dir,
778        "Official solution at",
779        Some(grader_map),
780        WriteBinTo::path("bin/official_solution"),
781    );
782    if official_solutions.len() > 1 {
783        let paths = official_solutions
784            .iter()
785            .map(|s| s.name())
786            .collect::<Vec<_>>();
787        bail!("Multiple official solutions found: {:?}", paths);
788    }
789    let official_solution = official_solutions.pop().map(Arc::new);
790    debug!("Detected output generator: {:?}", official_solution);
791    let output_directory = task_dir.join("output");
792    Ok(move |tc: TestcaseId| -> OutputGenerator {
793        if let Some(solution) = official_solution.as_ref() {
794            OutputGenerator::Custom(solution.clone(), vec![])
795        } else {
796            OutputGenerator::StaticFile(output_directory.join(format!("output{}.txt", tc)))
797        }
798    })
799}

801/// Parse the task components relative to the batch task type.
802fn parse_batch_task_data(task_dir: &Path, grader_map: Arc<GraderMap>) -> Result<TaskType, Error> {
803    let mut checkers = find_source_file(
804        task_dir,
805        vec!["check/checker.*", "cor/correttore.*"],
806        task_dir,
807        "Checker at",
808        None,
809        WriteBinTo::WithoutExtension,
810    );
811    if checkers.len() > 1 {
812        let paths = checkers.iter().map(|s| s.name()).collect::<Vec<_>>();
813        bail!("Multiple checkers found: {:?}", paths)
814    }
815    let checker = checkers
816        .pop()
817        .map(|mut c| {
818            // Always copy the custom checker.
819            c.copy_exe();
820
821            // Link the checker statically. This makes sure that it will work also outside this machine.
822            c.link_static();
823
824            Checker::Custom(Arc::new(c))
825        })
826        .unwrap_or(Checker::WhiteDiff);
827
828    let official_solution = detect_output_generator(task_dir.to_path_buf(), grader_map)
829        .context("Failed to detect output generator")?;
830    let official_solution = match official_solution(0) {
831        gen @ OutputGenerator::Custom(_, _) => Some(gen),
832        _ => None,
833    };
834    Ok(TaskType::Batch(BatchTypeData {
835        output_generator: official_solution,
836        checker,
837    }))
838}

840/// Parse the task components relative to the communication task type.
841fn parse_communication_task_data(
842    task_dir: &Path,
843    yaml: &TaskYAML,
844) -> Result<Option<TaskType>, Error> {
845    let mut managers = find_source_file(
846        task_dir,
847        vec!["check/manager.*", "cor/manager.*"],
848        task_dir,
849        "Communication manager at",
850        None,
851        WriteBinTo::WithoutExtension,
852    );
853    if managers.len() > 1 {
854        let paths = managers.iter().map(|s| s.name()).collect::<Vec<_>>();
855        bail!("Multiple managers found: {:?}", paths);
856    }
857    let mut manager = if let Some(manager) = managers.pop() {
858        manager
859    } else {
860        return Ok(None);
861    };
862
863    // Always copy the manager.
864    manager.copy_exe();
865
866    // Link the manager statically. This makes sure that it will work also outside this machine.
867    manager.link_static();
868
869    Ok(Some(TaskType::Communication(CommunicationTypeData {
870        manager: Arc::new(manager),
871        num_processes: yaml.num_processes.unwrap_or(1),
872        user_io: yaml.user_io,
873    })))
874}

886/// Normalize and validate the content of the subtask name.
887fn cleanup_subtask_name(id: &str) -> Result<String, Error> {
888    let id = id.trim();
889
890    let fail = |err| Err(anyhow!("'{}' is not a valid identifier: {}", id, err));
891
892    // Normalize the identifier to avoid similar but different characters.
893    let normalized = id.nfkc().collect::<String>();
894
895    if normalized.is_empty() {
896        return fail("must be non-empty");
897    }
898    if normalized.starts_with('-') {
899        return fail("must not start with a dash (-)");
900    }
901    for ch in normalized.chars() {
902        if ch == '*' {
903            return fail("must not contain asterisks (*)");
904        }
905        if ch == '?' {
906            return fail("must not contain question marks (?)");
907        }
908        let category = GeneralCategory::of(ch);
909        if !VALID_SUBTASK_NAME_CHARACTER_CATEGORIES.contains(&category) {
910            return fail(&format!(
911                "contains an invalid character '{}' ({})",
912                ch,
913                ch.escape_default()
914            ));
915        }
916    }
917    Ok(normalized)
918}
